# Purple Lord: Path of Choices | Фиолетовый Лорд: Путь Выбора

2D-платформер на MonoGame о выборе пути в IT-технологиях.

## Описание | Description

Игра рассказывает о Фиолетовом Лорде — персонаже, который путешествует по мирам IT-технологий, собирая знания и делая выбор в пользу специализации.

**Основная идея:** Нельзя охватить необъятное. Нужно сделать выбор и двигаться дальше.

## Структура проекта | Project Structure

```
PurpleLordPlatformer/
├── Content/              # Текстуры, звуки, шрифты
├── Core/                 # Ядро игры (Game1, GameState, Camera)
├── Entities/             # Игровые объекты
│   ├── Player/           # Игрок (Фиолетовый Лорд)
│   ├── Enemies/          # Враги
│   └── Objects/          # Объекты (платформы, двери, предметы)
├── Levels/               # Уровни
│   ├── Level1_FrontendForest/    # Лес Фронтенда
│   ├── Level2_BackendBadlands/   # Пустоши Бэкенда
│   ├── Level3_NeuralNebula/      # Туман Нейросетей
│   └── Bonus_JuniorOffice/       # Офис Джун
├── Managers/             # Менеджеры (сцены, контент, ввод, аудио, эффекты)
├── Scenes/               # Сцены (меню, уровни)
├── Systems/              # Системы (физика, анимация, пост-обработка)
├── Effects/              # Эффекты (частицы)
└── UI/                   # Интерфейс (меню, HUD)
```

## Уровни | Levels

### Уровень 1: Лес Фронтенда (Frontend Forest)
- **Визуал:** Яркие цвета, платформы в виде div-блоков
- **Враги:** Баги с глазами-иконками браузеров
- **Технологии:** HTML, CSS, JavaScript, React, Vue, Angular, TypeScript, SASS

### Уровень 2: Пустоши Бэкенда (Backend Badlands)
- **Визуал:** Тёмные тона, серверные стойки, логи на фоне
- **Враги:** Серверы, кружки кофе
- **Технологии:** Go, Python, Node.js, Java, PostgreSQL, Redis, MongoDB, Docker, Kubernetes, gRPC

### Уровень 3: Туман Нейросетей (Neural Network Nebula)
- **Визуал:** Абстрактные формы, исчезающие платформы, бинарный снег
- **Враги:** Нейроны-призраки
- **Технологии:** TensorFlow, PyTorch, Keras, Scikit-learn, OpenCV, NLP, GPT

### Бонусный уровень: Офис Джун (Junior Office)
- **Визуал:** Офис с кубиклами, IDE-иконки
- **Враги:** Кружки кофе, дедлайны
- **Технологии:** Git, Debugging, Unit Tests, Code Review, Agile/Scrum

## Механики | Mechanics

### Способности игрока
- **Прыжок** — базовый прыжок
- **Двойной прыжок** — метафора "попытки охватить необъятное"
- **Фокус (Focus Mode)** — замедление времени для осмотра и решения головоломок

### Сбор предметов "Знаний"
- На уровнях разбросаны предметы технологий
- При сборе показывается краткое описание
- 100% сбор недостижим (метафора невозможности знать всё)

### Эффект расфокусировки
- При долгом бездействии экран искажается
- Снимается активацией "Фокуса" или достижением цели

## Запуск | How to Run

## Архитектура | Architecture

Игра следует принципам чистой архитектуры:

- **Entity-Component-System (ECS)** — для игровых объектов
- **Менеджеры** — централизованное управление ресурсами
- **Сцены** — разделение на меню и уровни
- **События** — слабая связанность компонентов

## Расширение | Extending

### Добавление нового уровня
1. Создайте папку в `Levels/`
2. Унаследуйтесь от базового класса `Level`
3. Переопределите методы создания уровня
4. Добавьте в меню выбора уровней

### Добавление новой технологии
1. Добавьте в `KnowledgeItem.cs` новый тип
2. Создайте предмет на уровне
3. Добавьте описание технологии
